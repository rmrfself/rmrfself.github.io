Java中的工具类-Map接口

分类: JVM 发布于: 2015-09-12 21:40

Java语言中最重要的存贮形式: 基于hash值的散列桶的实现。它是二维形式下的仓库实现。

Map的特征

  • 没有重复的键值(key)

  • key 与 value是一对一的关系

  • 一个Map包含三种视图: Set类型的keys， Collection类型的values, 或者也可以看做 Set类型的 key-values 映射。

  • treeMap支持排序， Hashmap实现类没有排序。

HashMap的特征

  • 具备Map的基本特征

  • 允许空操作

  • 没有线程安全保证

  • Hashmap的性能取决于2个因素：初始容量和load factor(容量负载报警因子)

  • HashMap 的put实现方法解析
// final类型的方法

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        // 如果table 为空， 则初始化
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;
        // hash值和数组长度n进行 `与`， 返回n中的某个数值。
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
        else {
            Node<K,V> e; K k;
            // 如果 map中存在这个值
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                e = p;
            // 如果当前值不是普通的Node，而是转化后的TreeNode    
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else {
                // 发生碰撞的情况: key不同，但是hashcode相同
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        p.next = newNode(hash, key, value, null);
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }
                    // 存储成功
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }

潜在问题是同一个hashcode下的对象如何存储?

  • threshold(容量阈值)以下以linkedlist 存储

  • 阈值以上转化为treenode，以树🌲的结构存储使得读取效率提升为O(log(N))

WeakHashMap的实现

  • 非线程安全

  • 默认初始容量 2^4, 最大容量 2^30

  • Java对象引用的四种形式

  • 强引用

Object object = new Object(); 
String str = “hello”;

常规的声明赋值形式，强 体现在这种初始化形式不会被GC回收，无论发生什么紧急情况。

  • 软引用(SoftReference)

用来描述一些有用但不是必须的对象。

  • 弱引用

一个回收周期内会清除的对象。

  • 虚引用

任何时候都可能被回收。

HashTable 与 HashMap

  • 不存储空值

  • 与hashmap一样，依赖于hashcode，equals的实现

  • 它的常规方法默认增加了 线程同步操作。

  • 基本行为与hashMap一样

  • Dictionary 的子类

SortedMap 及其实现类TreeMap

  • 基于红-黑树(动态平衡树)

  • 基于key的排序，key需实现comparable接口

  • 增删改查具备tree的特征，复杂度log(n)
